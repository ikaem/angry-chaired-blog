Implement environment variables into the React app

I wanted to have environment variables in the app to be able to seamlessly switch between development and production api URLs, and not having to change those manually when I prepare the app for production deployment.
For instance, I want my const ENDPOINT to point to “localhost:5000 while I develop the app, and to onlineapiaddress.herokuapp.com when the app is already deployed. And I don’t want to switch these things manually.

React apps created with create-react-app have support for environment variables ready out of the box. An .env.production and .env.developoment files need to be placed in the project root, and any key inside these with a name pattern of “REACTAPP“ makes its value available as an environment variable.
Here, we don’t have that.

Important thing that is, well, important apparently, is that client app cannot really use custom environment variables in a way that a Node.js app can. Node.js is an environment on which its app runs, and which can provide these variables to the app. A client app runs in a browser, and we cannot set our custom variables into someone else’s browser.
For this reason, environment variables for client apps need to be embedded into code during the build time.
That’s as far as my understanding of it goes.

So how do we do this?

  • We need dotenv installed from NPM
  • We need to create files that hold data to be used as environment variables
  • We need to instruct Webpack to define the excution environment for every bundle it builds
  • We need to extract data from these files and embedd it into our Webpack bundle as environment variables, depending on the environment in which the app is being executed

So let’s do it.

For this reason, we should make sure these are added to the .gitignore file, so as not to be commited to Github.

Deploy backend

It is time to deploy the backend. Normally, with JS, I would just push the app to Heroku repository and be done with it. This time though, we have a TS project.

The idea is to push the TS project to Heroku, and have Heroku make the JS build and then serve it.

First thing to do is to disable calling nodemon with the start script. Instead, we want to call Node on the server.js file.

Next, we want Heroku to create the production build once it finishes installing dependencies of the project.
We do so by adding a postInstall script to package.json, and calling tsc in it. This will compile all TS data in src folder, and output it to dist folder:
